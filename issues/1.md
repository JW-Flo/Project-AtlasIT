### Overview
Establish initial relational schema (tenants, users, policies, workflows, connectors, audit_events, secrets_meta) and a repeatable migration mechanism.

### Goals
- Minimal tables + indexes to support auth, policy, workflow MVP.
- Forward-only production migrations; safe local dev down migrations.

### Non-Goals
- Multi-region replication strategy.
- Advanced partitioning/sharding.

### Initial Schema
- tenants(id UUID PK, name TEXT UNIQUE, created_at TIMESTAMPTZ DEFAULT now())
- users(id UUID PK, tenant_id UUID FK->tenants(id) ON DELETE CASCADE, email CITEXT, attributes JSONB, created_at TIMESTAMPTZ)
  Index: users(tenant_id, email) UNIQUE WHERE email IS NOT NULL
- policies(id UUID PK, tenant_id UUID FK->tenants, name TEXT, definition JSONB, created_at TIMESTAMPTZ)
  Index: policies(tenant_id, name) UNIQUE
- workflows(id UUID PK, tenant_id UUID FK->tenants, name TEXT, definition JSONB, created_at TIMESTAMPTZ)
  Index: workflows(tenant_id, name) UNIQUE
- connectors(id UUID PK, tenant_id UUID FK->tenants, type TEXT, config JSONB (encrypted-at-rest wrapper), created_at TIMESTAMPTZ)
  Index: connectors(tenant_id, type)
- audit_events(id BIGSERIAL PK, tenant_id UUID FK->tenants, actor_user_id UUID NULL, type TEXT, payload JSONB, hash_prev BYTEA, hash_curr BYTEA, created_at TIMESTAMPTZ DEFAULT now())
  Index: audit_events(tenant_id, created_at DESC)
  Index: audit_events(hash_curr) UNIQUE
- secrets_meta(id UUID PK, tenant_id UUID FK->tenants, name TEXT, key_ref TEXT, created_at TIMESTAMPTZ)
  Index: secrets_meta(tenant_id, name) UNIQUE

### Migrations
- Numbered files 0001_, 0002_ etc. Checksums recorded.
- Drift detection aborts startup.
- Local rollback scripts for first 2 versions only.

### Failure Modes
- Apply failure: abort with clear log & metric increment.
- Drift: abort and instruct manual intervention.

### Observability
- Logs: migration start/end, duration, version, status.
- Metrics: migration_apply_seconds (histogram), migration_error_total (counter), migration_version_gauge.

### Security & Data Integrity
- Enforce row-level tenant_id FKs.
- No plaintext secrets in connectors.config (only encrypted blobs).

### Acceptance Criteria
[ ] SQL migration files merged.
[ ] Re-running migrations is idempotent (no changes).
[ ] Drift detection test passes.
[ ] Basic unit/integration test applies migrations in ephemeral DB.
[ ] Documentation section added (schema overview).
