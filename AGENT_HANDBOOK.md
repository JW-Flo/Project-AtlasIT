# Agent Handbook

## Quick Reference

### Commit Standards
```bash
# Format
<type>: <description>
<type>(<scope>): <description>

# With evidence tag
ops(ci): Add drift scan (EV-0001)
```

### Evidence Emission
```bash
# Emit evidence artifact
npm run emit:evidence -- "CONTROL-ID" "pass" '{"key":"value"}'

# Aggregate all evidence
npm run evidence:aggregate
```

### Drift Detection
```bash
# Scan for framework drift
npm run drift:scan

# View results
cat artifacts/DRIFT-SCAN-*.json
```

### Routing Simulation
```bash
# Simulate PR routing
npm run simulate:routing -- pr-metadata.json

# With deterministic seed
npm run simulate:routing -- pr-metadata.json "custom-seed"
```

### OPA Policy Tests
```bash
# Run OPA tests
npm run opa:test

# Test specific policy
./scripts/run-opa.sh input.json
```

## Workflow Integration

### CI Pipeline
1. Build & test
2. Drift scan
3. OPA policy validation
4. Evidence aggregation
5. Artifact upload

### PR Workflow
1. PR opened → Metadata extracted
2. Routing decision computed
3. Labels applied (severity, agent)
4. Comment added with decision JSON
5. Agent assignment notification

### Evidence Management
- All control checks emit to `/artifacts`
- `INDEX.json` generated by aggregation
- Schema validation via OPA
- Compliance mappings in evidence metadata

## Common Tasks

**Add new control check:**
1. Implement check logic
2. Emit evidence via `emit-evidence.ts`
3. Add control to `NIST_CONTROLS` if applicable
4. Update OPA policies if needed

**Modify routing rules:**
1. Edit `ops/agent-router-worker/rules.json`
2. Update version number
3. Test with simulation
4. Deploy worker

**Handle drift:**
1. Run drift scan
2. Review missing files/directories
3. Restore from framework
4. Re-scan to verify
